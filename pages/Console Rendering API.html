Console Rendering API

I tried to make this API that lets me draw pixels to the console. I wanted to make a game.

Unfortunately, it is a horrendous mess that combines two ideas that are incompatible:

  1. Console Rendering
  2. Dithering


In short, consoles only change colours when they change. What this means is that, instead of having a colour for each character, it has uses the same colour as the previous character until it differs.

This also means that it is bad at continously changing colours. Horizontal stripes are quick, but vertical stripes aren't.

A 480p display takes 40 milliseconds to render when paired with dithering. Unfortunately, that means that the fastest it can smoothly display at is less than 25fps.

Now, you may be wondering: Why dither in the first place?

Well, the fastest way to print to the console is to use a Windows exclusive function. That function only lets us use 16 colours, but it is much faster than the good old Console.Write().
